ℹAbout This Project
What it does
Reads a config.yaml file and enforces desired system state — idempotently. Run it twice and the second run makes zero changes. Supports packages, users, file permissions, systemd services, and cron jobs.
Why it's relevant
Built to demonstrate skills directly aligned with Linux automation engineering — Python scripting, Bash, YAML-driven configuration, and familiarity with Ansible-style idempotent provisioning patterns used in real systems integration work.
